Create a new type

Define the name of the type. Must not contain spaces.

By convention, type names start with a capital letter for each word. For example:

  • Customer
  • FirstName
Type name

Provide a namespace for your type.

Normally, this contains the business domain that the type relates to. If it's a specialised type for a specific application, you might want to include the application name. Often includes the company name too.

  • com.acme.customers
  • com.acme.invoicing.invoicePlatform
Type namespace
Define what type of data this is. Either inherit from a basic type (such as String, Date or Boolean), or from another type in your schema to provide a more specialised version of that type.
